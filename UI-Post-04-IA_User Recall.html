Usability Errors Solutions Examples

Feature, Search & Browse

This pattern consists of having three elements in the main page of the site or app: a featured piece of content, a search box and a list of items or categories to browse.

This is commonly used when a site/app offers a long list of items that can be browsed and searched (Search & Browse, respectively), and when you want your incoming users to engage with something interesting right as they enter into the site or app (the Featured element)

News Stream

This pattern consists in showing time-sensitive items in a reverse chronological list, with the latest items at the top. The content updates dynamically, and allows combination of different sources or people in a single stream. It is advised to use this pattern when your site offers more than one communication channel such as blogs, email, news sites, or any medium of a similar nature. Some platforms allow "owning" a communication channel.

Picture Manager

Through the use of thumbnails, item views, and a browsing interface, create a familiar structure for managing photos, videos, and other pictorial items such as photos, video clips, drawings, and elements of the sort. This pattern is used quite commonly in softwares that allow the user to organize and/or edit the type of content previously mentioned.

Dashboard

This concept consists on arranging data displays into a single information-dense page that is regularly updated, while showing users relevant, interactive information and let them customize their display as necessary. This pattern is common in sites that deal with an incoming flow of information from something (web servers, news, airline flights, among others), at the same time that the user would find useful to monitor that information.

Canvas + Palette

This pattern consists in the placement of a distinctive palette next to a blank canvas, in order for the user to click on the palette buttons to create objects on the canvas. This is used when creating graphical editors of any kind, particularly when the end goal is creating new objects and arranging them in a virtual space.

Wizard

This concept is essentially leading the user through a set of tasks in a prescribed order with the aid of the interface. Often times, the nature of these guided tasks are relatively complicated for the user, and that's why (presumably, and most of the time) the user prefers to hand over the control of what happens in that process (to an extent).

Settings Editor

This concept consists of providing an easy-to-find, self-contained page or window where users can change settings, preferences and/or properties. It is advised to divide these settings in tabs or pages (often times these are simultaneously categories) if there's a large number of properties that the user can change.

Alternative Views

The whole page is fitted in about the 60% of the horizontal screen size available, causing the content of the web site to feel unnecesarily cluttered and saturated respect to space. This can be countered by distributing the content around the whole area of the screen.
